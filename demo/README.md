# Demo recordings

This directory holds the [VHS](https://github.com/charmbracelet/vhs) tape scripts
that generate the animated GIFs shown in the project README.

VHS is Charm's terminal-GIF recorder: it reads a `.tape` script — a plain list of
commands like `Type`, `Enter`, `Sleep` — drives a real headless terminal through
those keystrokes, and renders the session to a GIF. That makes the demos
reproducible and reviewable in version control instead of hand-recorded.

## Contents

| File | Purpose |
| ---- | ------- |
| `setup-demo-env.sh` | Builds `rulem-demo` and provisions a self-contained sandbox (config + sample rule repo + a project dir). Idempotent. |
| `main-demo.tape` | Hero demo: save a rule file (file picker with live markdown preview → rename → save). Renders `gifs/demo.gif`. |
| `import-demo.tape` | Import a rule from the central repo into a project as `AGENTS.md`. Renders `gifs/import.gif`. |
| `settings-demo.tape` | Read-only tour of the settings menu and a repository's actions. Renders `gifs/settings.gif`. |

## Prerequisites

- **VHS**: `go install github.com/charmbracelet/vhs@latest`
- **ttyd** and **ffmpeg** on your `PATH` — VHS shells out to both to render.
  On macOS install them via Homebrew (`brew install ttyd ffmpeg`), MacPorts, or nix.

Without `ttyd` and `ffmpeg` VHS can still `validate` a tape (syntax check) but
cannot render a GIF.

## Regenerating the GIFs

Run everything from the repository root:

```bash
# 1. Build the demo binary and (re)provision the sandbox.
./demo/setup-demo-env.sh

# 2. Render each GIF (needs ttyd + ffmpeg installed).
vhs demo/main-demo.tape       # -> demo/gifs/demo.gif
vhs demo/import-demo.tape     # -> demo/gifs/import.gif
vhs demo/settings-demo.tape   # -> demo/gifs/settings.gif

# Syntax-check a tape without rendering:
vhs validate demo/main-demo.tape
```

Each tape sets `XDG_CONFIG_HOME` to `demo/sandbox/config` in a hidden setup block,
so rulem reads the sandbox config (which points at the sample `demo/sandbox/rules-repo`)
instead of your real configuration. Nothing outside `demo/` is touched.

## What is and isn't committed

- **Committed**: the tape scripts, `setup-demo-env.sh`, this README, and — once
  rendered — the GIFs under `demo/gifs/` (the project README embeds them, so
  GitHub needs the actual files in the repo).
- **Git-ignored** (regenerated by the setup script, never committed):
  `demo/rulem-demo`, `demo/sandbox/`.

After rendering, commit the updated GIFs:

```bash
git add demo/gifs/*.gif && git commit -m "docs: refresh demo GIFs"
```
